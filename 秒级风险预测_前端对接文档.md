# 秒级风险预测器 — 前端对接文档

> 更新日期: 2026-05-05  
> 后端版本: v3.1  
> 新增功能: 秒级快速风险预警（消除10秒AI分析真空期）

---

## 一、功能概述

新增"**秒级快速风险预测**"通道，与现有的报警和AI分析**并行运行**：

| 通道 | WebSocket type | 频率 | 用途 |
|------|---------------|------|------|
| 异常报警 | `alarm` | 有异常才推 | 已超标，立即通知 |
| **秒级预警** | `fast_risk` | **每秒每设备** | 趋势预测：还没超标但可能即将超标 |
| AI深度分析 | `ai_analysis` | 10秒/设备 | 根因分析+建议+知识库引用 |

**核心价值**：在异常真正发生前，通过趋势+速度+波动综合判断，提前预警。

---

## 二、WebSocket 消息格式

### 连接地址（不变）

```
ws://localhost:8080/ws/alarm
```

### 新增消息: `fast_risk`

```json
{
  "type": "fast_risk",
  "deviceCode": "DEV00002",
  "batchNo": "B20260505001",
  "processType": "Temp",
  "currentValue": 139.2,
  "riskFast": 0.72,
  "riskLevel": "MEDIUM",
  "deviation": 0.88,
  "slope": 0.0823,
  "volatility": 0.1245,
  "etaSeconds": 22.5,
  "target": 137.0,
  "minThreshold": 135.0,
  "maxThreshold": 140.0,
  "timestamp": 1746441600000
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | String | 固定 `"fast_risk"` |
| `deviceCode` | String | 设备编码 |
| `batchNo` | String | 当前批次号 |
| `processType` | String | 工序类型 |
| `currentValue` | Double | 当前传感器实时值 |
| `riskFast` | Double | **快速风险分 0~1**（核心指标） |
| `riskLevel` | String | `LOW` / `MEDIUM` / `HIGH`（已防抖） |
| `deviation` | Double | 偏离度 0~1（离目标多远，1=到边界） |
| `slope` | Double | EWMA趋势斜率（正=上升，负=下降） |
| `volatility` | Double | 30秒滚动标准差（波动大小） |
| `etaSeconds` | Double | **预计触线时间（秒）**，-1=安全无趋势 |
| `target` | Double | 工艺目标值 |
| `minThreshold` | Double | 工艺下限 |
| `maxThreshold` | Double | 工艺上限 |
| `timestamp` | Long | 事件时间戳(ms) |

---

## 三、风险分级

| riskLevel | riskFast 范围 | 含义 | 前端建议 |
|-----------|-------------|------|---------|
| `LOW` | 0 ~ 0.6 | 运行正常 | 绿色/灰色，正常显示 |
| `MEDIUM` | 0.6 ~ 0.8 | 趋势危险，即将逼近阈值 | 黄色闪烁，显示ETA倒计时 |
| `HIGH` | 0.8 ~ 1.0 | 极度危险，几秒内可能超标 | 红色闪烁+声音提示 |

**防抖机制**（后端已处理）：
- 连续3秒满足中/高风险才升级
- 连续5秒回落才降级
- 前端收到的 `riskLevel` 已经是稳定值，不会闪烁

---

## 四、ETA（预计触线时间）使用指南

| etaSeconds | 含义 | 前端展示建议 |
|-----------|------|------------|
| `-1` | 没有朝边界移动（安全）| 显示 "—" 或不显示 |
| `> 60` | 较远 | 显示 ">60s" |
| `30~60` | 需关注 | 显示具体秒数，黄色 |
| `15~30` | 即将触线 | 显示倒计时，橙色 |
| `< 15` | 紧急！ | 红色倒计时+闪烁 |

### 5.3 WebSocket 消息处理

```javascript
// 在现有 WebSocket onmessage 中增加分支
ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  
  switch (data.type) {
    case 'alarm':
      // 已有：异常报警通知
      handleAlarm(data);
      break;
      
    case 'fast_risk':
      // 新增：秒级风险预警
      handleFastRisk(data);
      break;
      
    case 'ai_analysis':
      // 已有：10秒AI深度分析
      handleAiAnalysis(data);
      break;
  }
};

function handleFastRisk(data) {
  // 按 deviceCode 更新对应设备卡片
  const device = deviceMap[data.deviceCode];
  if (!device) return;
  
  device.currentValue = data.currentValue;
  device.riskFast = data.riskFast;
  device.riskLevel = data.riskLevel;
  device.deviation = data.deviation;
  device.slope = data.slope;
  device.etaSeconds = data.etaSeconds;
  
  // 趋势箭头
  device.slopeArrow = data.slope > 0.01 ? '↑' 
                    : data.slope < -0.01 ? '↓' : '→';
  
  // 风险颜色
  device.riskColor = data.riskLevel === 'HIGH' ? 'red'
                   : data.riskLevel === 'MEDIUM' ? 'orange' : 'green';
}
```

### 5.4 流量控制建议

每秒8台设备 = 8条消息/秒，数据量不大。但如果只展示3台设备（白名单期间），可以在前端过滤：

```javascript
const SHOW_DEVICES = ['DEV00001', 'DEV00002', 'DEV00006'];

function handleFastRisk(data) {
  if (!SHOW_DEVICES.includes(data.deviceCode)) return;
  // ...
}
```

## 六、三种消息对比总结

| 维度 | alarm | fast_risk | ai_analysis |
|------|-------|-----------|-------------|
| 触发条件 | 值已超标 | 每秒每设备 | 批次运行中+10秒窗口 |
| 延迟 | <1秒 | <1秒 | 10秒+LLM耗时 |
| 信息量 | 报警事实 | 数值+趋势+ETA | 原因+建议+知识库 |
| 前端用途 | 弹窗/通知 | 实时仪表盘 | 详细分析面板 |
| 是否持久化 | ✅ MySQL | ❌ 仅推送 | ✅ MySQL+Redis |

---

## 七、注意事项

1. **fast_risk 消息量较大**（每秒8条），前端不要把每条都存历史，只保留最新一条/设备
2. **riskLevel 已做防抖**，前端直接用，不需要再做平滑
3. **etaSeconds = -1** 表示安全（没有趋势逼近边界），注意处理
4. **slope 可能为负**（如温度在下降），前端用正负判断方向
5. 生产停止后后端不会推送 fast_risk（有批次校验）
6. 当前白名单期间只推 DEV00001/DEV00002/DEV00006
