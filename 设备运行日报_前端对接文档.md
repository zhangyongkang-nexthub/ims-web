# 设备运行日报 + AI 维保建议 — 前端对接文档

> 后端已完成，需前端实现页面  
> 后端基地址: `http://localhost:8080`  
> 建议路由: `/ai/device-report`

---

## 一、接口清单（共 3 个）

| 序号 | 方法 | 路径 | 说明 | 耗时 |
|------|------|------|------|------|
| 1 | GET | `/device-report/daily?date=2026-05-03` | 生成指定日期日报 | 5~15秒（首次调AI） |
| 2 | GET | `/device-report/daily/today` | 生成今日日报 | 5~15秒（首次调AI） |
| 3 | GET | `/device-report/daily/latest` | 获取最近缓存日报 | <100ms |

> ⚠️ 接口 1 和 2 首次调用会请求 DeepSeek AI 生成报告，耗时较长，**务必加 loading 状态**。  
> 生成后会缓存到 Redis，后续调接口 3 可快速获取。

---

## 二、请求参数

### 2.1 生成指定日期日报

```
GET /device-report/daily?date=2026-05-03
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| date | String | 否 | 昨日 | 格式: `yyyy-MM-dd` |

### 2.2 生成今日日报

```
GET /device-report/daily/today
```

无参数，基于今日已有的 IoT 数据生成。

### 2.3 获取最近缓存日报

```
GET /device-report/daily/latest
```

无参数。如无缓存返回：
```json
{ "code": 0, "msg": "暂无缓存的日报，请先生成" }
```

## 四、字段说明

### 4.1 顶层字段

| 字段 | 类型 | 说明 |
|------|------|------|
| reportDate | String | 报告日期，格式 `yyyy-MM-dd` |
| generateTime | String | 报告生成时间 |
| deviceSummaries | Array | 8台传感器的当日运行摘要（固定8条） |
| alarmTopDevices | Array | 当日告警最多的设备排名（0~N条） |
| aiDailyOverview | String | AI 生成的产线全局日评文字 |
| maintenanceSuggestions | Array | AI 维保建议列表（0~N条） |

### 4.2 deviceSummaries 字段

| 字段 | 类型 | 说明 | 前端用途 |
|------|------|------|---------|
| deviceCode | String | 传感器编码 DEV0000x | 表格列 |
| deviceType | String | 类型: BRIX/TEMP/VOLUME/WEIGHT/VISION/TORQUE/PRESSURE/PH | 图标/标签 |
| equipName | String | 所属设备名称 | 表格列 |
| sampleCount | Long | 当日采样总数 | 表格列 |
| mean | Double | 当日均值 | 表格列 |
| stdDev | Double | 当日标准差 | 表格列 |
| outOfRangeRate | Double | 超标率(%)，如 0.08 表示 0.08% | 表格列，超标高的标红 |
| alarmCount | Integer | 当日告警次数 | 表格列，>0 标红 |
| healthScore | Double | 健康评分 0~100 | **重点展示**，进度条/环形图 |
| trend | String | `UP`/`DOWN`/`STABLE` | 趋势箭头 ↑↓→ |
| runningDays | Long | 累计运行天数 | 表格列 |

### 4.3 alarmTopDevices 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| deviceCode | String | 设备编码 |
| deviceType | String | 设备类型 |
| alarmCount | Integer | 当日告警次数 |
| topAlarmMsg | String | 最近一条告警信息 |

### 4.4 maintenanceSuggestions 字段

| 字段 | 类型 | 说明 | 前端用途 |
|------|------|------|---------|
| deviceCode | String | 设备编码 | 关联设备 |
| equipName | String | 设备名称 | 卡片标题 |
| urgency | String | `HIGH`/`MEDIUM`/`LOW` | **标签颜色**: 红/橙/绿 |
| suggestion | String | AI 维保建议内容 | 卡片正文 |
| suggestedTime | String | 建议保养日期 | 卡片副标题 |
| reason | String | AI 判断依据 | 折叠详情 |

### 5.2 交互逻辑

```
页面加载
  │
  ├─ 先调 GET /device-report/daily/latest
  │    ├─ 有缓存 → 直接渲染
  │    └─ 无缓存(code=0) → 显示"暂无日报，请选择日期生成"
  │
  ├─ 用户点击 [生成日报] 按钮
  │    ├─ 显示 loading（"AI 正在分析，请稍候..."，约5-15秒）
  │    ├─ 调 GET /device-report/daily?date=选择的日期
  │    ├─ 成功 → 渲染页面
  │    └─ 失败 → 提示错误
  │
  └─ 快捷按钮: [生成今日日报]
       └─ 调 GET /device-report/daily/today
```

### 5.3 前端样式要点

| 元素 | 说明 |
|------|------|
| healthScore | 建议用环形进度条，≥90 绿色，70-89 橙色，<70 红色 |
| trend | `UP` → 绿色 ↑，`DOWN` → 红色 ↓，`STABLE` → 灰色 → |
| urgency 标签 | `HIGH` → 红色 Tag，`MEDIUM` → 橙色 Tag，`LOW` → 绿色 Tag |
| aiDailyOverview | 带背景卡片，AI 图标，支持换行 |
| maintenanceSuggestions | 卡片列表，按 urgency 排序（HIGH 在前），reason 放折叠面板 |
| alarmCount | >0 时红色字体加粗 |
| outOfRangeRate | >0.1% 时橙色，>0.5% 时红色 |

### 5.4 日期选择器

- 默认日期：昨日
- 可选范围：最近 30 天
- 格式：`yyyy-MM-dd`
- 不可选未来日期

## 六、API 调用示例 (axios)

```javascript
import request from '@/utils/request'  // 封装的 axios 实例

// 获取最近缓存的日报
export function getLatestReport() {
  return request.get('/device-report/daily/latest')
}

// 生成指定日期日报
export function generateDailyReport(date) {
  return request.get('/device-report/daily', {
    params: { date }  // 格式: '2026-05-03'
  })
}

// 生成今日日报
export function generateTodayReport() {
  return request.get('/device-report/daily/today')
}
```
